ログイン

LP を編集（通常）: 「site_key」＝アップロードしたフォルダ名、パスワード＝ Builder が示す初期パスワード （custom/cms_credentials.json と同じ）。初回はパスワード変更のあと編集できます。
管理者のみ: site_key を空にして ID（例: lp-admin）でログイン。

LP: /cms/api/site-login.php · 管理者: /cms/api/login.php

初回: パスワードを変更

簡易初期パスワードのままでは編集できません。12文字以上の新パスワードを設定してください。

編集するLPを選択

手動: 上で選んで「このLPで作業」。クライアントから /cms/admin/?site_key=（作成した site_key） を開くと、ログイン後（または既にログイン中なら起動直後）に 自動で POST select-site し編集画面へ遷移します。サーバは GET だけでは切り替えません。

コンテンツ編集

左で入力 → 右のプレビューが近い雰囲気で更新されます（本番 LP のヒーロー帯に準拠したスタイル）。保存後、公開ページは custom/cms_page_state.json 経由で反映（/cms/overlay-apply.js 読込が必要）。

画像

アップロードで …/custom/ に保存し、下のファイル名に自動入力されます。

文言（ページ内の代表テキスト）

本番用ディレクトリ（アップロード＝オリジナル → 生成＝_lp_publish）

① 編集用の …/<site_key>/ にアップロードした元一式を _lp_original に固定。② CMS で保存した内容と custom/ から _lp_publish へ本番用を再生成。②は何度も可能（リドゥ＝毎回オリジナル＋今の DB 上の編集から再生成）。 URL 振り分けは server/docs/examples/lp-site-root.htaccess.example を参照。
